The distinction between discussing a concept and naming a specific string is one that most technical writing blurs. You can write about functions in general using prose, but the moment you're talking about the `sort.Slice` function in Go — that specific name, that specific package — you need the monospace treatment.

The visual shift does two things. It signals to the reader that this is a literal token they might type, search for, or look up. And it separates the writer's words from the language's words, which matters enormously when you're writing about a language whose vocabulary overlaps with ordinary English.

This is why `return` looks different from the word return. One is an instruction the reader should internalize; the other is a command the computer will execute.
